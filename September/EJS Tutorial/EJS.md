### Server Side Rendring (SSR)

Also Called Dynamic Rendring or Dynamic websites

In this everthing works and handles on the Server Side.

DataBase -> Backend Application -> Templates  (here comes the EJS) -> JSON data  ->  HTML, CSS & JS files

These all things than goes to the Clint Side (Browser) 


1. Browser will send request to Server
2. Server will process it.
3. Server will send Fully rendered HTML

Pros. 

1. No load on Personal Computers.
2. Better SEO (Search Engine Optimization)
3. Good For Static websites.

Cons. 

1. More load on server
2. Slower Navigation

Ex : Next.js, EJS


### Client side Rendreing (CSR)

Also called Web API Application

In this everthing works and handles on the ClientSide.


1. Browser will send request to server
2. Sends Minimal Code (HTML + JS)
3. Browser Downloads the files.
4. Page appears after the Js execution.

Pros. 

1. Rich Interactivity and Smooth Transition
2. Less Server load
3. Great for Web Apps ( Like Chat Apps or Social Media Apps or Dashboard )

Cons.

1. Slower First Load
2. SEO Issues

Ex : React, Angular, Vue.
